# Project Instructions

## Session Startup
At the start of every session or after a context clear:
1. Read `CURRENT_STATE.md` to understand the current state of the project
2. Run `gh issue list --state open` to get the current list of open GitHub issues

## Progress Tracking
After completing any task or fix cycle:
1. Update `CURRENT_STATE.md` to reflect the current project status
2. Append a dated entry to `PROGRESS.md` to maintain a historical track record of what was changed and why
